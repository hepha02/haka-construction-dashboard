# HAKA Construction 공사비 관리 시스템 기획 기준

이 프로젝트는 HAKA Construction의 매장 공사비를 관리하는 실무형 관리자 시스템이다.

## 핵심 목표

- 매장별 공사 진행 상태와 공사비를 한 화면에서 확인한다.
- 결제 신청, 승인, 반려, 증빙 미비, 중복 의심을 관리한다.
- 업체/계좌, 매장, 공사비, 진열장 원가, 견적서, 계약서, 은행 이체 파일을 단계적으로 자동화한다.
- Supabase를 실제 데이터베이스로 사용하고, GitHub Pages로 공개 확인 주소를 유지한다.

## 메뉴 구조

- 대시보드
- 엑셀 업로드
- 결제 신청
- 업체/계좌 관리
- 매장별 공사 관리
- 진열장 원가 배분
- 견적서 생성
- 계약서 생성
- 은행 이체 파일 생성
- 관리자 설정

## 대시보드

KPI 카드:

- 완료된 매장
- 진행중인 매장
- 승인된 공사비
- 실제 사용액
- 대기중인 결제
- 증빙 미비
- 중복 의심
- 평균 평당 원가

하단 영역:

- 최근 결제 신청
- 매장 공사 현황
- 주요 협력업체
- 통계 요약

## 기능 개발 순서

1. 현재 공개 배포와 Supabase 연결 유지
2. 결제 신청 입력 및 DB 저장
3. 중복 의심 경고
4. 업체/계좌 관리
5. 매장별 공사 관리
6. 엑셀 업로드와 컬럼 검증
7. 견적서/계약서 생성
8. 은행 이체 파일 생성
9. 로그인/권한 관리

## 디자인 기준

- 오래된 ERP 스타일 금지
- 흰색 사이드바와 부드러운 회색 앱 배경
- 카드형 KPI와 테이블
- 두꺼운 테두리, 과한 파란색 사이드바, 이모지 아이콘 금지
- 데스크톱 업무 화면 중심, 모바일에서도 깨지지 않게 유지
- 금액은 KRW 오른쪽 정렬
- 상태 배지는 부드러운 색상 사용

## 상태 배지

- 승인: green
- 신청: amber
- 반려: red
- 진행중: blue
- 완료: green
- 미착공: gray
